This wiki describes several ways to get the next/previous page. However, if you are looking for a way to only display a specific set of posts or want to display X posts per page, then you can check out the Phile Paginator plugin.

Since Phile 1.2

Let's assume you content folder looks like this:

content/
|-- 404.md
|-- blog
|   |-- 1.md
|   `-- 2.md
|-- index.md
|-- projects
    |-- 1.md
    `-- 2.md

and you want a navigation in the blog pages. First you should group all pages by subfolder and sort by date (a usual meta-tag for blog pages): in your config.php set

$config['pages_order']     = 'page.folder:asc meta.date:asc meta.title:desc';

Then use in your twig template for the blog pages:

{% for page in pages %}
    {% if current_page.title == page.title %}
        {% if page.previousPage is defined and page.previousPage.folder == "blog" %}
            <a href="{{ page.previousPage.url }}"> Previous </a>
        {% endif %}

        {% if page.nextPage is defined and page.nextPage.folder == "blog" %}
            <a href="{{ page.nextPage.url }}"> Next </a>
        {% endif %}
    {% endif %}
{% endfor %}

Directory hierarchy (pre Phile 1.2)

Phile doesn't provide a good general solution for previous/next pagination (yet). This wiki shows you several workarounds to provide a previous/next pagination on your Phile site. Feel free to add another solution if you have implemented one.

This paginator relies on the directory hierarchy. For example, if your Phile site has the following hierarchy:

content/
|-- 404.md
|-- blog
|   |-- 1.md
|   |-- 2.md
|   |-- 3.md
|   |-- 4.md
|   `-- 5.md
|-- index.md
|-- projects
    |-- 1.md
    `-- 2.md

Then you can just take a look at the url of the current page and replace the number (the ID) by its predecessor/successor:

<ul class="pager">
  <li><a href="{{ base_url }}/blog/{{ current_page.url | split('/') | last - 1}}">&laquo;</a></li>
  <li><a href="{{ base_url }}/blog/">More posts</a></li>
  <li><a href="{{ base_url }}/blog/{{ current_page.url | split('/') | last + 1}}">&raquo;</a></li>
</ul>

Pro

  • Takes into account in which directory the posts are stored (e.g. project posts vs. blog posts).

Cons

  • Directory hierarchy assumes the post with the highest ID is the latest post (they are most of the cases)
  • There's no possibility to check whether the URL is valid, so the next button of the latest post will redirect you to the 404 page, likewise for the previous button for the first post.
