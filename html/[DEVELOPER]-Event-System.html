One of the main feature of Phile is new event system, which replace the hook system of pico.

In the core Phile triggers a lot of events, and for every event you can register your plugin to get informed. Depending on the event data is passed along an can be manipulated by the plugin.

You can find a list of all triggered events on the startpage of the demo pages after installing Phile.

The easiest way to subscribe to events is using the $events property in your plugin:

class Plugin extends \Phile\Plugin\AbstractPlugin {

    protected $events = ['after_load_content' => 'doStuff'];

    protected doStuff($data) { 
        // do something
    }

That's it. Now when the after_load_content event happens doStuff() is called.
